# Multiwiki

**Multiwiki** - это динамическое веб-приложение, платформа для создания и хостинга тематических вики-энциклопедий. Подобно Fandom, сервис позволяет любому зарегистрированному пользователю создать собственную вики на интересующую его тему - например, вики по игре, по вселенной книги или выдуманной расе. Внутри каждой такой вики пользователи могут наполнять её статьями, устанавливать связи между ними, загружать медиафайлы и организовывать навигацию. Таким образом, Multiwiki объединяет множество независимых вики-проектов, формируя сообщества вокруг узких интересов.

**Multiwiki** подойдёт всем, кто хочет создать свою тематическую энциклопедию. Это фанаты игр, фильмов и книг, которым нужен удобный справочник по любимой вселенной. Писатели и ведущие настольных игр смогут структурировать материалы собственных миров. А ещё платформа пригодится студентам, преподавателям и небольшим командам для совместной работы над базами знаний.



# Ключевой функционал

* Возможность зарегистрироваться и создать неограниченное количество вики-сайтов, каждый со своим уникальным названием и тематикой.
  
* Создание, редактирование, удаление и версионирование статей внутри конкретной вики.

* Присвоение статьям категорий, создание внутренних ссылок между статьями для формирования связного гипертекста.

* Возможность иллюстрировать статьи, загружая изображения и другие файлы в хранилище вики.

* Разделение прав на «Владельца» вики (полный контроль), «Редактора» (создание и изменение статей) и «Читателя» (только просмотр).



# Структура приложения

* Главная страница платформы (/) - публичная, представляет собой лендинг с информацией о сервисе, списком популярных вики и призывом к регистрации или созданию своей вики.

* Аутентификация (/login, /register) - раздел для входа, регистрации. Доступен только неавторизованным пользователям.

* Профиль пользователя (/profile) - приватная страница, где пользователь может изменить ник, аватар, пароль и другие настройки.



* Создание новой вики (/create) - приватный, форма для создания собственной вики (название, описание, адрес :name).

* Редактирование вики (/wiki/:name/edit) - приватный, форма для редактирования существующей вики (доступна автору или редакторам).

* Главная страница вики (/wiki/:name) - публичная, представляет собой стартовую страницу конкретной вики. Содержит описание вики, популярные статьи.


* Создание категории (/wiki/:wiki_name/category/create) - приватный, форма для создания новой категории (доступна редакторам).

* Редактирование категории (/wiki/:wiki_name/category/:category_name/edit) - приватный, форма для изменения категории.

* Список категорий статей (/wiki/:wiki_name/category) - публичная, перечисляет все категории, созданные в рамках данной вики, с кратким описанием.

* Страница категории (/wiki/:wiki_name/category/:category_name) - публичная, показывает все статьи, принадлежащие выбранной категории.



* Создание статьи (/wikis/:wiki_name/category/:category_name/article/create) - приватный, форма для создания новой статьи в текущей вики (доступна редакторам).

* Редактирование статьи (/wikis/:wiki_name/category/:category_name/article/:article_name/edit) - приватный, форма для редактирования существующей статьи (доступна автору или редакторам).

* Просмотр статьи (/wikis/:wiki_name/category/:category_name/article/:article_name) - публичная, отображает содержимое статьи.



* Создание форума (/wiki/:wiki_name/forum/create) - приватный, форма для создания нового обсуждения (доступна всем авторизованным).

* Редактирование форума (/wiki/:wiki_name/forum/:id/edit) - приватный, форма для редактирования существующей форума (доступна автору).

* Список форумов (обсуждений) (/wiki/:wiki_name/forum) - публичная, показывает все созданные в данной вики темы для обсуждений, отсортированные по дате последнего сообщения.

* Конкретная тема форума (wiki/:wiki_name/forum/:id) - публичная, отображает выбранное обсуждение с вопросами и ответами пользователей.


# Технологический стек
# Frontend
* React.js - обеспечивает компонентный подход, позволяя создавать переиспользуемые блоки (карточки статей, посты форума, формы). Виртуальный DOM гарантирует высокую производительность при динамическом обновлении контента. Большая экосистема и гибкость упрощают интеграцию с другими библиотеками.
  
* React Router - стандартное решение для навигации в React. Позволяет легко реализовать вложенные маршруты (/:name/wiki/article/:id), защиту приватных страниц и обработку динамических параметров. Поддерживает удобные хуки для доступа к истории и параметрам URL.

* State-менеджмент - используется встроенный Context API для глобальных данных.

# Backend
* Java - строгая типизация снижает количество ошибок на этапе компиляции и облегчает поддержку кода. Высокая производительность и многопоточность позволяют обрабатывать множество одновременных запросов (чтение статей, поиск, комментарии).

* Spring Boot - ускоряет разработку благодаря автоконфигурации, встроенному серверу и обширному набору стартеров. Spring Security обеспечит гибкую аутентификацию (JWT, OAuth2) и разграничение прав (создатель вики, редактор, читатель). Spring Data JPA упрощает работу с реляционной базой, минимизируя шаблонный код.

* MySQL - реляционная СУБД с поддержкой ACID-транзакций, необходимая для целостности данных при множественных операциях (создание статей, добавление категорий, форумные обсуждения). Позволяет эффективно моделировать связи между сущностями (пользователи, вики, статьи, категории) и обеспечивает быстрый поиск благодаря индексации текстовых полей.

# Инфраструктура
* Локальная разработка - на текущем этапе приложение запускается на локальной машине. Это удобно для отладки и тестирования без затрат на хостинг.

* Блок приветствия: Краткое описание сайта и кнопка создать свой вики.

# Аутентификация "/login" "/register"
* Шапка: Название сайта "Multiwiki", выбор языка, кнопки войти, зарегистрироваться
* Форма входа: поля email и пароля, кнопка "войти"
* Форма регистрации: поля почты, логина, пароля и подтверждения пароля и кнопка "зарегитрироваться"
* Футер: контакты и информация о сервисе
* Динамический конетнт: Отправка данных форм на бэкенд, вывод ошибок валидации и ошибок от запросов на бэкенд

# Профиль "/profile" (Приватный)

* Динамический конетнт: Отправка данных форм на бэкенд, вывод ошибок валидации и ошибок от запросов на бэкенд

# Схема данных и API

# Модели данных
# Users - пользователь.
* id - уникальный идентификатор.
* username - имя пользователя.
* email - почта.
* password - хеш пароля.
* role - роль пользователь (администратор или пользователь).
* createdAt - дата создания пользователя.
* updatedAt - дата обновления пользователя.

# Wikis - вики.
* id - уникальный идентификатор.
* name - уникальное английское название для ссылок.
* backgroundImage - изображение фона вики.
* userId - автор вики.
* createdAt - дата создания вики.
* updatedAt - дата обновления вики.

# Categories - категория.
* id - уникальный идентификатор.
* name - уникальное английское название для ссылок.
* wikiID - айди вики.
* userId - автор категории.
* createdAt - дата создания категории.
* updatedAt - дата обновления категории.

# Articles - статья.
* id - уникальный идентификатор.
* name - уникальное английское название для ссылок.
* wikiID - айди вики.
* categoryId - айди категории.
* userId - автор статьи.
* createdAt - дата создания статьи.
* updatedAt - дата обновления статьи.

# Posts - пост.
* id - уникальный идентификатор.
* title - заголовок поста.
* body - тело поста.
* userId - автор поста.
* createdAt - дата создания поста.
* updatedAt - дата обновления поста.

# Images - изображения.
* id - уникальный идентификатор.
* userID - автор изображения.
* filename - имя файла или путь.
* url - публичный URL.
* createdAt - дата добавления изображения.

# Translations - переводы.
* id - уникальный идентификатор.
* translatableType - тип сущности(wiki, category, article).
* translatableId - айди сущности.
* locale - код языка.
* title - заголовок.
* body - тело.
* createdAt - дата создания перевода.

* id - уникальный идентификатор.

# Ключевые эндпоинты API
# Users
* POST /register - регистрация нового пользователя.
* POST /login - вход в систему.

* GET /users - список пользователей (только для администраторов).
* GET /users/{id} - получение информации о пользователе.
* PUT /users/{id} - обновление данных пользователя (только владелец).
* DELETE /users/{id} - удаление пользователя (только владелец или администратор).

# Wikis
* GET /wikis - список вики.
* GET /wikis/{name} - получение информации о вики.
* PUT /wikis/{name} - обновление данных вики.
* POST /wikis - создание новой вики.

# Categories
* GET /wikis/categories - получение всех категорий.
* GET /wikis/{name}/categories/{name} - получение информации о категории.
* PUT /wikis/{name}/categories/{name} - обновление данных категории.
* POST /wikis/{name}/categories - создание новой категории.

# Articles
* GET /wikis/{name}/categories/articles - получение всех статей.
* GET /wikis/{name}/categories/{name}/articles/ получение всех статей определённой категории.
* GET /wikis/{name}/categories/articles/{name} - получение информации о статье.
* PUT /wikis/{name}/categories/articles/{name} - обновление данных статьи.
* POST /wikis/{name}/categories/articles - создание новой статьи.

# Posts
* GET /wikis/{name}/posts - получение всех форумов.
* GET /wikis/{name}/posts/{id} - получение информации о форуме.
* PUT /wikis/{name}/posts/{id} - обновление данных форума.
* POST /wikis/{name}/posts - создание нового форума.

# Images
* POST /images/upload - загрузка изображения
* DELETE /images/{id} - удаление изображения (только владелец или администратор).

# Translations
* GET /wikis/{name}/translations - получение всех переводов вики.
* GET /wikis/{name}/translations/{local} - получение перевода вики.
* POST /wikis/{name}/translations/{local} - создание перевода вики.
* PUT /wikis/{name}/translations/{local} - обновление перевода вики.

* GET /wikis/{name}/categories/{name}/translations - получение всех переводов категории.
* GET /wikis/{name}/categories/{name}/translations/{local} - получение перевода категории.
* POST /wikis/{name}/categories/{name}/translations/{local} - создание перевода категории.
* PUT /wikis/{name}/categories/{name}/translations/{local}- обновление перевода категории.

* GET /wikis/{name}/categories/{name}/articles/{name}/translations - получение всех переводов статьи.
* GET /wikis/{name}/categories/{name}/articles/{name}/translations/{local} - получение перевода статьи.
* POST /wikis/{name}/categories/{name}/articles/{name}/translations/{local} - создание перевода статьи.
* PUT /wikis/{name}/categories/{name}/articles/{name}/translations/{local}- обновление перевода статьи.
  
# Messages
* GET /wikis/{name}/posts/{id}/messages - получение всех сообщений форума.
* POST /wikis/{name}/posts/{id}/messages - создание нового сообщения под форумом.
* DELETE /messages/{id_message} - удаление сообщения.
